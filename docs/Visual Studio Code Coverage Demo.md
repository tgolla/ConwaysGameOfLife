# Visual Studio Code Coverage (Demo)

## A Discussion on Unit Testing

1. Open Visual Studio and select the **ConwayGameOfLife** example.
2. Create a branch. Open **Git -> Manage Branches** window. Right click the **main** branch and select **New Local Branch From...** then enter the branch name **CodeCoverage** and click **Create**.
3. Open the Solution Explorer window to discuss how this demo is looking at unit test specifically on the service layer which contains the business rules. This should be a discussion on each of the layers.
   - The API/UI layer which while it can be tested with a mocked service layer, lends itself more to functional testing with Playwright.
   - The data layer build on Entity Framework is a well tested repository class from Microsoft that doesn't need to be test, except for added methods.
   - The the service layer containing the business rule which we will focus on.


## Running Code Coverage

1. Start by running all tests **Test -> Run All Test**.

2. In Test Explorer drill down into the **ConwaysGameOfLifeServicesUnitTests** and double click the `Seed_WithValidLivePoints_CreatesBoardAndLivePoints` test to open the `ConwaysGameOfLifeServicesUnitTests` file.

3. Now run code coverage **Test -> Analyze Code Coverage for All Test**.

4. In the Code Coverage Results window drill down through the hierarchy to `conwaysgameoflife.data.dll` -> `ConwaysGameOfLife.Data.Migration` and note how this code has no code coverage , but that it is also not necessary as the code it generated by Microsoft's Entity Framework which is already test. Remember you don’t test things like simple ***getters, framework code, or external systems***.

5. Next drill down through the hierarchy to `conwaysgameoflife.services.dll` -> `ConwaysGameOfLifeService` -> `ConwaysGameOfLifeService(ILogger<ConwaysGameOfLifeService> logger, IConfiguration configuration, ConwaysGameOfLifeApiDbContext conwaysGameOfLifeApiDbContext)` and double click to open the .CS file.

6. Note to code coverage for the constructor in the 70's and point out the covered lines highlighted in light blue and the partial coverage in gray.

7. Now change the colors to something more user friendly by going to **Tools** -> **Options...** In the left menu select **Environment** -> **Fonts and Colors**, then click on **Fonts and Colors** to open the Options dialog.

8. In the **Display items:** box scroll down to `Coverage Not Touched Area` and change the **Item background:** to **Red**. Select  `Coverage Partially Touched Area` and change the **Item background:** to **Yellow**. Select  `Coverage Touched Area` and change the **Item background:** to **Lime**. Click **OK** to save the new settings. 

9. Return to the `ConwaysGameOfLifeService.cs` file and note the lines are now highlighted in green and yellow. Scroll down to `Transition(Guid boardId, uint iterations)` and note the red.

10. Next remove the line highlighting by selecting the **Show Code Coverage Coloring** icon in the Code Coverage Results and click the down arrow to unselect **Line Coloring**.

11. Return to the class constructor method `ConwaysGameOfLifeService()` and announce that we are now going to write a unit test for the partially covered code. Discuss naming and the variation that works best with .NET and C# files broken in to classes (*MethodName_StateUnderTest_ExpectedBehavior*). From the code construct the test name `ConwaysGameOfLifeService_MissingConwaysGameOfLifeSettings_InvalidOperationException`.

12. Open the `ConwaysGameOfLifeServicesUnitTests.cs` file and locate the `Seed_WithValidLivePoints_CreatesBoardAndLivePoints()` method. About the test method add `[Test]` and the method `public void ConwaysGameOfLifeService_MissingConwaysGameOfLifeSettings_InvalidOperationException`().

13. Now lets play with the AI Copilot Chat. Open the Copilot Chat and type
    ```Show me some test code that covers the ConwaysGameOfLifeService constructor InvalidOperationException```

14. The AI will attempt to fill in the test, it may or may not  be correct and it will aways be different. The following is test code based on existing test.

    ````c#
       [Test]
        public void ConwaysGameOfLifeService_MissingConwaysGameOfLifeSettings_ThrowsInvalidOperationException()
        {
            // Arrange
            var conwaysGameOfLifeDbContext = CreateInMemoryDbContext();
            var loggerMock = new Mock<ILogger<ConwaysGameOfLifeService>>();
            var configuration = new ConfigurationBuilder().Build();
    
            // Act & Assert
            Assert.Throws<InvalidOperationException>(() =>
            {
                var ConwaysGameOfLifeServiceInstance = new ConwaysGameOfLifeService(loggerMock.Object, configuration, conwaysGameOfLifeDbContext);
            });
        }
    ````

    And this is something AI returned on a different day. Again similar, but different and it does work.

    ```c#
        [Test]
        public void ConwaysGameOfLifeService_MissingConwaysGameOfLifeSettings_InvalidOperationException
        { 
            // Arrange
            var conwaysGameOfLifeDbContext = CreateInMemoryDbContext();
            var loggerMock = new Mock<ILogger<ConwaysGameOfLifeService>>();
    
            // Create a configuration that does NOT contain the ConwaysGameOfLifeSettings section.
            var emptySettings = new Dictionary<string, string>();
    #pragma warning disable CS8620 // Argument cannot be used for parameter due to differences in the nullability of reference types.
            var configuration = new ConfigurationBuilder()
                .AddInMemoryCollection(emptySettings)
                .Build();
    #pragma warning restore CS8620 // Argument cannot be used for parameter due to differences in the nullability of reference types.
    
            // Act & Assert
            var ex = Assert.Throws<InvalidOperationException>(() => new ConwaysGameOfLifeService(loggerMock.Object, configuration, conwaysGameOfLifeDbContext));
                StringAssert.Contains("ConwaysGameOfLifeSettings section is missing or invalid", ex.Message);
        }
    ```

    

15. Once the test passes, rerun the code coverage **Test -> Analyze Code Coverage for All Test**. Now return to the `ConwaysGameOfLifeService.cs` file and point out how code coverage has changed to green.

## Demo Reset

1. Delete the **CodeCoverage** branch and check out **Main**. Open **Git -> Manage Branches** window. Right click the **main** branch and select checkout. Right click the **CodeCoverage** branch and select delete.
2. Reset code coverage colors by going to **Tools -> Options -> Environment -> Fonts and Colors** and click **Fonts and Colors**. Select **Coverage Not Touched Area**, **Coverage Partially Touched Area**, and **Coverage Touched Area** and reset the background color to **Default**.
3. Make sure coverage is set to color lines and display glyphs by running **Test -> Analyze Code Coverage** for all Test. In the **Code Coverage Results** window click the line dropdown icon and make sure **Line Coloring** and **Margin Glyphs** are both selected.
4. Clear all test results. **Test -> Clear All Test Results**
5. Close all windows. Close and reopen Visual Studio.
